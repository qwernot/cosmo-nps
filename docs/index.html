🖥️ 第一部分：总控部署

总控是整个系统的核心管理中枢，负责 Web 后台、用户管理、节点管理和隧道配置下发。

🚀 方式一：一键脚本部署（推荐）

脚本会自动完成以下所有操作：检测系统 → 安装 Docker（含阿里云镜像加速）→ 拉取镜像 → 交互式配置 → 启动服务 → 健康检查。

一键脚本交互式配置界面

1

在服务器上执行一键部署命令

SSH 登录到总控服务器，以 root 或 sudo 执行：

# 下载项目并执行一键部署脚本
git clone https://github.com/qwernot/tunnel-all.git
cd tunnel-all
bash deploy/quick-deploy.sh

脚本会交互式询问三个配置项：

提示说明留空默认值
公网 IP/域名服务器的公网地址，留空自动获取自动检测
管理员用户名后台登录账号admin
管理员密码后台登录密码随机生成 12 位强密码
2

等待自动完成

脚本会自动安装 Docker、配置镜像加速、拉取镜像、启动服务，最后进行健康检查。成功后输出：

==========================================
  ✓ 总控部署成功！
==========================================

  访问地址: http://1.2.3.4:8088
  用户名:   admin
  密码:     MyStr0ngP@ss

  常用命令:
    查看状态: docker compose ps
    查看日志: docker compose logs -f tunnel-stack
    重启服务: docker compose restart tunnel-stack
    停止服务: docker compose down
==========================================
✅ 部署完成！请妥善保存输出的账号密码，直接访问 Web 后台即可。跳到 第二部分：总控使用。

方式二：手动 Docker 部署

如果你已经安装了 Docker，或者需要自定义配置，可以手动部署。

  • 一台有公网 IP 的服务器（或内网可访问的机器）
  • 已安装 Docker 和 Docker Compose
  • 放行端口 8088/tcp（Web 后台）
1

创建部署目录

mkdir -p ~/tunnel-stack && cd ~/tunnel-stack
2

编写 docker-compose.yml

services:
  tunnel-control:
    image: darkver8/tunnel-all:latest
    container_name: tunnel-stack
    restart: unless-stopped
    network_mode: host
    environment:
      PUBLIC_ADDR: 192.168.6.64        # 替换为你的服务器公网 IP
      ADMIN_USER: admin                # 管理员用户名
      ADMIN_PASSWORD: change-this-password  # 管理员密码（务必修改！）
    volumes:
      - ./data:/app/data
环境变量说明：
PUBLIC_ADDR服务器公网地址，用于生成客户端配置中的连接地址
ADMIN_USER首次启动时自动创建的管理员用户名
ADMIN_PASSWORD管理员密码，请设置强密码
3

启动总控

docker compose up -d

查看启动日志确认是否正常运行：

docker compose logs -f tunnel-stack

看到类似 created bootstrap admin user "admin" 和 listening on :8088 即表示启动成功。

4

验证运行

curl http://127.0.0.1:8088/healthz

返回 ok 即表示总控正常运行。访问 http://你的IP:8088 即可打开 Web 后台。

创建隧道

点击左侧导航 「🔌 隧道」 进入隧道管理页面。

隧道管理页面示意图

隧道支持以下模式：

模式说明是否需要端口是否需要域名
tcpTCP 端口转发✅ 远程端口❌
udpUDP 端口转发✅ 远程端口❌
socks5SOCKS5 代理✅ 远程端口❌
httpHTTP 域名转发❌✅ 域名
httpsHTTPS 域名转发❌✅ 域名
💡 示例说明：创建一个 TCP 隧道，远程端口 10001，本地目标 127.0.0.1:8080。用户访问 节点IP:10001 即会被转发到用户本地机器的 8080 端口。

查看客户端配置

点击左侧导航 「⚙️ 配置」，选择用户和节点，可以查看：

  • NPS 命令：原生 npc 启动命令
  • 统一客户端：tunnel-client 启动命令（推荐）
  • NPS 密钥：用户的 NPS VerifyKey

⚡ 第三部分：节点部署

节点是实际承载隧道流量的服务器，运行 NPS 引擎并接收总控的配置下发。

前置要求

  • 一台有公网 IP 的服务器
  • 已安装 Docker 和 Docker Compose（或使用一键部署脚本）
  • 在总控后台已创建节点，并获取了 节点 Token
  • 放行以下端口
端口用途
18089/tcp节点配置接口（建议只允许总控访问）
18024/tcpNPS Bridge
18025/tcpNPS TLS Bridge
9080/tcpNPS HTTP Proxy
9443/tcpNPS HTTPS Proxy
10000-20000/tcp+udp用户隧道端口

方式一：一键部署（推荐）

总控后台内置了节点一键部署功能。脚本会优先通过 Docker 启动容器节点；若无 Docker，则自动下载二进制并注册为 Systemd 系统服务。

1

在总控后台获取部署命令

进入 「🖥️ 节点」 页面，找到目标节点，点击 「部署」 按钮。

一键部署弹窗示意图

2

在节点服务器上执行

SSH 登录到节点服务器，粘贴执行复制的命令即可。脚本会自动完成所有配置。

方式二：Docker 手动部署

如果你已经安装了 Docker，或者需要自定义端口配置，可以手动部署。

1

创建部署目录

mkdir -p ~/tunnel-agent && cd ~/tunnel-agent
2

编写 docker-compose.yml

services:
  tunnel-agent:
    image: darkver8/tunnel-port:latest
    container_name: tunnel-agent-edge8
    restart: unless-stopped
    network_mode: host
    entrypoint: ["/usr/local/bin/tunnel-agent"]
    environment:
      CONTROL_URL: http://192.168.6.64:8088     # 总控地址
      NODE_ID: edge-8                           # 节点 ID（与后台一致）
      NODE_TOKEN: paste-node-token-here         # 粘贴节点 Token
      AGENT_API_ADDR: ":18089"                  # Agent API 端口
      NPS_BRIDGE_PORT: 18024
      NPS_TLS_BRIDGE_PORT: 18025
      NPS_HTTP_PORT: 9080
      NPS_HTTPS_PORT: 9443
    volumes:
      - ./data:/app/data
3

启动节点

docker compose up -d

查看日志确认连接成功：

docker compose logs -f tunnel-agent

看到 synced node=edge-8 users=X tunnels=X 即表示节点已成功同步配置。

✅ 验证：回到总控后台 「🖥️ 节点」 页面，该节点状态应显示为 在线。

以下为桌面端/本地部署方式：

Windows GUI 客户端（推荐）

Windows 客户端运行效果示意图

1

构建/下载客户端

# Windows 构建（含高清图标，无 CMD 黑框）
go build -ldflags="-H windowsgui -s -w" -o tunnel-client.exe ./cmd/tunnel-client
2

启动客户端

直接双击 tunnel-client.exe，不会弹出任何黑框窗口，直接缩入系统托盘。

浏览器会自动打开 http://127.0.0.1:18090 管理面板。

3

配置连接

在 Web 面板中填入：

总控地址http://你的总控IP:8088
用户名管理员为你创建的账号
密码对应密码

点击连接后，客户端会自动登录总控、获取隧道配置、连接对应节点。

4

高级功能

功能说明
开机自启托盘右键 → 开启「开机自启动」
安装为系统服务tunnel-client.exe -service-install
卸载系统服务tunnel-client.exe -service-uninstall
静默启动tunnel-client.exe -silent（不打开浏览器）

Linux 守护进程

1

构建

# Linux 交叉编译（无需 C 编译器）
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags="-s -w" -o tunnel-client-linux ./cmd/tunnel-client
2

运行

# 带 Web GUI 启动（自动打开浏览器）
./tunnel-client-linux

# 命令行直接启动（无 GUI）
./tunnel-client-linux -server http://192.168.6.64:8088 -user alice -password alice123

命令行静默模式（通用）

1

直接传参启动

tunnel-client -server http://192.168.6.64:8088 -user alice -password alice123

不传 -user -password 时自动进入本地 Web 面板模式。
