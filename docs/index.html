Cosmo NPS

NPS 多节点统一管理系统 · 总控、节点、客户端三角色部署教程

系统架构

总控后台 darkver8/cosmo-nps:latest

只负责 Web 登录、用户、节点、隧道、端口池、域名池和配置下发，不承载业务流量。

节点服务 darkver8/cosmo-nps-node:latest

运行 tunnel-agent 和 NPS，负责实际穿透入口、HTTP/HTTPS 代理和用户隧道端口。

用户客户端 darkver8/cosmo-nps-client:latest

运行 tunnel-client。用户只连接总控，客户端自动读取隧道并连接对应节点。

三个镜像已经拆分构建：总控、节点、客户端不再是同一个镜像打三个标签。

一、总控部署

总控只需要开放后台端口，默认是 8088/tcp，部署时可以通过 CONTROL_PORT 自定义。

一键脚本

git clone https://github.com/qwernot/cosmo-nps.git
cd cosmo-nps
sudo bash deploy/quick-deploy.sh

静默部署

sudo PUBLIC_ADDR=192.168.6.64 \
  CONTROL_PORT=8088 \
  ADMIN_USER=admin \
  ADMIN_PASSWORD='change-this-password' \
  NONINTERACTIVE=1 \
  bash deploy/quick-deploy.sh

Compose 示例

services:
  cosmo-nps:
    image: darkver8/cosmo-nps:latest
    container_name: tunnel-stack
    restart: unless-stopped
    ports:
      - "${CONTROL_PORT:-8088}:${CONTROL_PORT:-8088}"
    environment:
      PUBLIC_ADDR: 192.168.6.64
      CONTROL_PORT: ${CONTROL_PORT:-8088}
      ADMIN_USER: admin
      ADMIN_PASSWORD: change-this-password
    volumes:
      - ./data:/app/data

访问地址：

http://总控IP:${CONTROL_PORT:-8088}

二、节点部署

先在总控后台创建节点，复制节点 Token。节点服务器推荐使用后台生成的一键部署命令，也可以手写 compose。

后台生成的一键命令

curl -fsSL "http://总控IP:8088/api/agent/bootstrap?id=节点ID&token=节点TOKEN" | bash

Compose 示例

services:
  cosmo-nps-node:
    image: darkver8/cosmo-nps-node:latest
    container_name: tunnel-agent-edge8
    restart: unless-stopped
    network_mode: host
    entrypoint: ["/usr/local/bin/tunnel-agent"]
    environment:
      CONTROL_URL: http://总控IP:8088
      NODE_ID: edge-8
      NODE_TOKEN: paste-node-token-here
      AGENT_API_ADDR: ":18089"
      NPS_BRIDGE_PORT: 18024
      NPS_TLS_BRIDGE_PORT: 18025
      NPS_HTTP_PORT: 9080
      NPS_HTTPS_PORT: 9443
    volumes:
      - ./data:/app/data
端口用途
18089/tcp节点配置接口，建议只允许总控访问
18024/tcpNPS bridge
18025/tcpNPS TLS bridge
9080/tcpNPS HTTP proxy
9443/tcpNPS HTTPS proxy
10000-20000/tcp/udp用户隧道端口池，按你的节点配置调整

三、客户端部署

用户客户端只需要填写总控地址、用户名和后台密码。

Docker 客户端

services:
  cosmo-nps-client:
    image: darkver8/cosmo-nps-client:latest
    container_name: tunnel-client
    restart: unless-stopped
    network_mode: host
    entrypoint: ["/usr/local/bin/tunnel-client"]
    environment:
      CONTROL_URL: http://总控IP:8088
      TUNNEL_USER: dark
      TUNNEL_PASSWORD: change-this-password

Windows 客户端

dotnet build ./cmd/tunnel-client-gui/TunnelClientGui.csproj -c Release
go build -ldflags="-H windowsgui -s -w" -o tunnel-client.exe ./cmd/tunnel-client

命令行客户端

tunnel-client -server http://总控IP:8088 -user dark -password change-this-password

五、本地构建与推送

项目使用 Dockerfile 多 target 构建三个独立镜像。

go test ./...

docker build --target control -t darkver8/cosmo-nps:latest .
docker build --target node -t darkver8/cosmo-nps-node:latest .
docker build --target client -t darkver8/cosmo-nps-client:latest .

docker push darkver8/cosmo-nps:latest
docker push darkver8/cosmo-nps-node:latest
docker push darkver8/cosmo-nps-client:latest

也可以直接使用脚本：

bash build-and-push.sh
